# Backend Conference System

## Overview

Backend Conference System представляет собой учебный проект, реализованный в архитектуре микросервисов. Каждый сервис вынесен в отдельную ветку репозитория, что обеспечивает изоляцию логики, независимую разработку и удобство сопровождения.

Проект включает три самостоятельных сервиса:

- сервис управления конференциями  
- сервис управления сессиями  
- сервис управления отзывами  

Каждый сервис содержит собственный код, зависимости и конфигурацию.

---

## Architecture

Проект разделён на три микросервиса, каждый из которых отвечает за отдельный функциональный блок.

| Service            | Branch     | Description                                           |
|--------------------|------------|-------------------------------------------------------|
| ConferenceService  | conference | Управление конференциями, создание и редактирование данных |
| SessionService     | session    | Управление сессиями внутри конференций               |
| FeedbackService    | feedback   | Приём и обработка отзывов участников                 |

Каждый сервис включает следующие элементы:

- основной файл запуска `main.py`  
- файл окружения `.env`  
- файл зависимостей `requirements`  
- внутренние модули и вспомогательные компоненты
